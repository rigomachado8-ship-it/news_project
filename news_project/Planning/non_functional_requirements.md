# Non-Functional Requirements

1. The application should use MariaDB as the database engine.
2. The UI should be simple and easy to navigate.
3. Pages should load correctly for public and authenticated users.
4. Role permissions should be enforced securely.
5. The application should separate public article access from restricted article management features.
6. Code should follow PEP 8 and Django style conventions.
7. Views and major logic blocks should include comments for maintainability.
8. The project should be organized into clear apps, templates, and URL routes.
9. The system should provide a responsive and readable user experience.
10. The application should use environment variables where appropriate for configuration.
