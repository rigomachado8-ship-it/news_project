# Functional Requirements

1. Users can register through the web interface.
2. Users can log in and log out through the web interface.
3. Readers can browse approved articles.
4. Journalists can create articles.
5. Journalists can edit their own articles.
6. Editors can review pending articles.
7. Editors can approve or reject articles.
8. Authenticated users can manage subscriptions to publishers.
9. Authenticated users can manage subscriptions to journalists.
10. Authenticated users can view subscribed articles.
11. The system supports publishers and newsletters.
12. The API provides article listing and article detail endpoints.
13. The API provides a subscribed articles endpoint for authenticated users.
14. The API provides a pending articles endpoint for editors.
